Theme Integration Guide

This guide is for MyBB admins and theme developers who want to customize how the Credits plugin looks on their forum. It covers every CSS class, template variable, and rendered HTML structure so you can tailor the plugin to match any custom theme.

Who is this for?

  • Forum admins running custom or third-party MyBB themes
  • Theme developers building themes that integrate with Credits
  • Anyone who wants to override or extend the plugin's default styles

How Styling Works

The plugin's CSS is provided as a standalone file that you paste into your theme manually. This is the most reliable approach — no auto-injection issues, works on any hosting environment, and survives plugin updates.

Setup Steps

  1. Locate the STYLES.css file included in the release package.
  2. In your MyBB Admin CP, go to Themes & Templates → Your Theme → global.css.
  3. Paste the contents of STYLES.css at the end of your global.css.
  4. Save the stylesheet. The plugin's styles will now be active on your forum.

Customizing Styles

To customize the plugin's appearance, simply edit the CSS you pasted into global.css. The CSS includes variables at the top that control colors, spacing, and other visual properties — adjust these to match your theme. Since the styles live directly in your theme's stylesheet, you have full control over every rule.

Template Injections

The plugin modifies four core MyBB templates during activation. Understanding these injection points helps you control where plugin content appears.

Template Injection Point Variable Inserted
postbit After {$post['user_details']} {$post['credits_display']}
postbit_classic After {$post['user_details']} {$post['credits_display']}
member_profile Before {$footer} {$credits_profile}
header Before {$awaitingusers} {$credits_ad_header}

If your theme uses different template names

Some custom themes rename or restructure these templates. If the plugin's injections don't appear, check that your theme's postbit template still contains {$post['user_details']} and your header template contains {$awaitingusers}. If not, you'll need to manually add the plugin's template variables to your theme's templates.

Postbit Background

Users can purchase custom postbit backgrounds from the shop. The plugin supports three formats: solid colors, CSS gradients, and background images.

How It Renders

The plugin generates a <style> tag per post that targets the post's author panel using the post ID:

HTML Output 
<!-- Generated for each post with a custom background -->
<style>
  #post_{PID} .post_author {
    background-color: #1a1a2e !important;
  }
</style>

Supported Formats

Solid Color

Hex color codes

#1a1a2e

Generates: background-color: #1a1a2e !important;

CSS Gradient

linear, radial, or conic

linear-gradient(135deg, #667eea, #764ba2)

Generates: background: [value] !important;

image

Image URL

Any valid image path

images/postbit_bg.png

Generates: background: url(...) center/cover no-repeat !important;

Target Element

The plugin targets #post_{PID} .post_author. This is the standard MyBB postbit author panel. If your custom theme uses a different class name or structure for the author panel, the backgrounds won't apply.

Custom Theme Fix

If your theme does not use .post_author, you need to edit the plugin's rendering function. Open inc/plugins/credits/hooks.php around line 117 and change the CSS selector:

// Default - targets .post_author
$post['credits_bg_style'] = '<style>#post_' . $post_pid . ' .post_author { ' . $bg_css . ' }</style>';

// Change .post_author to your theme's class, e.g.:
// .post_author  -->  .author_info
// .post_author  -->  .postbit-left
// .post_author  -->  .post_sidebar

Styling Tips for Theme Developers

/* Make postbit backgrounds look good with rounded corners */
.post_author {
    border-radius: 8px;
    overflow: hidden;
    transition: background 0.3s ease;
}

/* Ensure text remains readable on custom backgrounds */
.post_author {
    text-shadow: 0 1px 2px rgba(0,0,0,0.3);
}

/* Add padding so background has room to breathe */
.post_author {
    padding: 12px;
}

/* If using dark backgrounds, ensure links stay visible */
.post_author a {
    color: #93c5fd;
}

Customizing Effects for Your Theme

Override any effect in your theme's stylesheet. For example, to change the gradient colors: .credits_fx_gradient a { background: linear-gradient(90deg, #your, #colors); }

Username Colors

Users can purchase a custom hex color that overrides their username's default color (including usergroup color).

HTML Structure

<span class="credits_ucolor" style="--credits-ucolor: #ff6b6b;">
    <a href="..." style="color: #0066cc;">Username</a>
</span>

CSS Mechanism

Uses a CSS custom property (--credits-ucolor) with !important to override both usergroup colors and inline styles:

.credits_ucolor,
.credits_ucolor * {
    color: var(--credits-ucolor) !important;
}

Important Note

The !important is necessary because MyBB applies usergroup colors as inline style="color: ..." attributes on username links. The CSS custom property approach is the cleanest way to override inline styles without JavaScript.

Awards & Badges

Data Format

Awards are stored as a JSON array of image paths in the user's credits_awards field:

["images/awards/gold.png", "images/awards/helper.png", "images/awards/veteran.png"]

Postbit HTML

<br />
<span class="credits_awards">
    <img src="images/awards/gold.png"
         alt="award"
         class="credits_award"
         style="vertical-align: middle; max-height: 16px; max-width: 16px; margin: 0 2px;" />
    <!-- repeated for each award -->
</span>

Booster Badge

Postbit HTML

Profile Section

The plugin adds a credits table to the member profile page, injected before the footer. It uses standard MyBB table markup.

Profile Table Structure

<table class="tborder">
  <tr><td class="thead" colspan="2">Credits</td></tr>

  <!-- Always shown -->
  <tr><td class="trow1">Balance</td><td class="trow1">{credits}</td></tr>
  <tr><td class="trow2">Rank</td><td class="trow2">#{rank}</td></tr>

  <!-- Conditional rows (only shown if user has these) -->
  {$credits_profile_icon_row}          <!-- User icon -->
  {$credits_profile_awards_row}        <!-- Awards display -->
  {$credits_profile_booster_row}       <!-- Active booster -->
  {$credits_profile_bg_row}            <!-- Background preview -->
  {$credits_profile_effect_row}        <!-- Username effect preview -->
  {$credits_profile_achievements_row}  <!-- Earned achievements -->
</table>

Since the profile section uses standard MyBB classes (.tborder, .thead, .trow1, .trow2), it will automatically inherit your theme's table styling.

JavaScript API

The plugin exposes a global Credits object loaded from jscripts/credits.js. Requires jQuery.

Credits.getBalance(callback)

Fetches the current user's credit balance via AJAX.

Credits.getBalance(function(balance) {
    console.log('User has ' + balance + ' credits');
});

Credits.refreshNavBalance()

Updates the credit count shown in the navigation bar link. Call this after any action that modifies the user's balance.

// After a purchase completes
Credits.refreshNavBalance();

Credits.initColorPicker()

Syncs color picker and hex text input fields in the shop. Called automatically on page load.

// Binds input[name="purchase_value"] (color) <--> input[name="purchase_value_hex"] (text)

AJAX Endpoint

The plugin registers an AJAX handler at:

POST xmlhttp.php?action=credits&operation=get_balance&my_post_key={key}
// Returns: { "success": true, "balance": 150 }

User Database Fields

The plugin adds these columns to the mybb_users table. These are the raw values used for rendering:

Column Type Example Value Used For
credits INT UNSIGNED 250 Current balance
credits_last_login_bonus INT 1707350400 Last login bonus timestamp
credits_icon VARCHAR(255) images/icons/star.png User icon path
credits_username_color VARCHAR(7) #ff6b6b Username color override
credits_awards TEXT ["img/a.png","img/b.png"] Award image paths (JSON)
credits_postbit_bg VARCHAR(255) linear-gradient(135deg, ...) Postbit background value
credits_username_effect VARCHAR(50) rainbow Active username effect
credits_referral_code VARCHAR(12) a3f8b2c1e9d0 Unique referral code
